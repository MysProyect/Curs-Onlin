Desde mi punto de vista creo que estás abordando el problema de la forma inversa, pues tu punto de referencia o inicial parecen ser las categorías:

Aunque no estoy de acuerdo con hacer consultas desde el controlador, pero para hacer más rápido el ejemplo, lo haré según tu estructura, asumiendo que tienes un modelo Categoria con la relación definida hacia otro modelo llamado Articulo:

$categorias = Categoria::with('articulos')->get();

return view('...', compact('categorias'));

En la vista (este código se puede mejorar y no está probado):

@foreach ($categorias as $categoria)

    {{ $categoria->nombre }}:

    @foreach ($categoria->articulos as $index => $articulo)

        @php $nombres[] = $articulo->nombre @endphp

    @endforeach

    @if (isset($nombres))
        {{ implode(', ', $nombres) }}
    @endif

@endforeach

Para que entiendas un poco más sobre las relaciones en Laravel, revisa: https://laravel.com/docs/5.4/eloquent-relationships
